# Loop Logic

> Core loop, meta loop, economy loops, and progression systems. The heartbeat of the game.

## Loop Hierarchy

```
Meta Loop (session-to-session progression)
└── Core Loop (per-session or per-round)
    ├── Action Loop (per-turn or per-second)
    └── Economy Loop (resource in/out flow)
```

## Loop Anatomy Template

Every loop note should answer:
1. **Input** — what does the player do?
2. **Feedback** — what does the game do immediately?
3. **Reward** — what changes over time?
4. **Tension** — what creates risk or decision points?
5. **Exit Condition** — how does this loop end or escalate?
